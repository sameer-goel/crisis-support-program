# 🕊️ Crisis Support Program

**AI-Powered Donor–Recipient Matching Platform**

A government-style web application that uses AI to connect people in crisis with verified volunteers and donors — matching needs to help in real time.

![Split Landing](https://img.shields.io/badge/Status-Live-brightgreen) ![Pages](https://img.shields.io/badge/Hosted-GitHub%20Pages-blue)

---

## 🖥️ Pages

| Page | Description |
|------|-------------|
| **index.html** | Split-screen landing — "In Crisis?" / "Want to Help?" |
| **ineedhelp.html** | Recipient flow — AI typing simulation, donor pool scan, criteria matching, profile reveal, meeting scheduler |
| **iwanttohelp.html** | Donor dashboard — real-time needs with fulfillment bars, quick-donate tiles, urgent requests feed |
| **volunteers.html** | Browse verified volunteers — search, AI search, category filters, profile cards |
| **matching.html** | AI Matching Engine visualization — real-time animated matching between people in need and helpers |
| **recycle360.html** | Bonus: Recycle 360 — government recycling initiative, scan/identify items, get disposal instructions |

## ✨ Features

- **AI Matching Simulation** — animated donor pool scanning with face avatars, criteria tick-through, time-slot search
- **Split-Screen Landing** — immersive 50/50 layout with distinct color gradients for each path
- **Donor Dashboard** — live needs bars (red/amber/green), 9 quick-donate categories, urgent request feed
- **Volunteer Directory** — ADPList-style cards with ratings, sessions, AI search toggle
- **Government Branding** — official header/footer, ministry naming, compliance badges, hotline bar
- **Fully Responsive** — stacks cleanly on mobile
- **Zero Dependencies** — pure HTML, CSS, and vanilla JavaScript

## 🛠️ Tech

- HTML5 / CSS3 / Vanilla JS
- No frameworks, no build step
- Avatar images via [i.pravatar.cc](https://i.pravatar.cc)
- Hosted on GitHub Pages

## 🚀 Run Locally

Just open `index.html` in a browser. No server needed.

```bash
open index.html
```
